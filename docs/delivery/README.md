# VerifyForGood Delivery Units

Delivery Units are coherent functional increments that can be released, enabled, adopted, demonstrated, or accepted end to end. They belong to one Initiative and contain one or more Build Units unless the level is intentionally collapsed.

## Required Metadata

- Delivery Unit ID
- Initiative
- Outcome delivered
- Release / enablement / adoption boundary
- Scope and non-goals
- Affected Domains and Architecture areas
- Build Units
- Dependencies and risks
- Acceptance criteria and validation
- Rollback or disablement posture
- Status and last reconciliation

## Active Records

- [`INIT-009 — Documentation and Traceability Baseline`](INIT-009.md)

A Delivery Unit is not a Roadmap phase. Roadmap placement may change without changing Delivery Unit identity.
